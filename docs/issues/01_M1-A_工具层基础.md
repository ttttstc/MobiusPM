# [M1-A] 工具层 Part 1:Excel 解析 + WorkItem + 稳定 itemId + SQLite 状态 + 幂等频控

## 目标
搭建项目骨架,落地"读表 → 标准化 WorkItem → SQLite 状态存储 → 安全边界(幂等/频控)"四件套。本 issue 完成后,工具层的"读"和"状态"维度全部就绪,可独立测试。

## 范围
- Python 项目初始化(`pm_agent/` 包、`requirements.txt`、`__main__.py` CLI 入口)
- 配置加载(`config/pm-agent.yaml`)
- `domain/`:
  - `work_item.py` — WorkItem dataclass
  - `status_mapper.py` — PRD §8 状态映射全表
  - `item_id.py` — 稳定 itemId(`sha1(来源 + "::" + normalize(原文))[:12]`)
- `tools/excel.py` — `read_excel()` tool:
  - 只读模式打开,运行后主表 mtime 不变
  - 解析 179 条 WorkItem(允许 ±5 偏差)
  - 多责任人字段拆分(`/、,，` 多分隔符)
  - 优先级映射 P0/P1/P2/Ignore
  - 标准状态映射
  - 失联检测:返回值带 `vanished_item_ids`(本次没扫到但 SQLite 有的)
- `memory/`:
  - `schema.sql` — 四张表(item_state / follow_up_log / decision_log / context_brief),按设计文档 §5
  - `store.py` — SQLite 封装,WAL 模式,初始化时自动建表
- `tools/state.py`:
  - `query_state(item_id?)` tool — 查事项当前状态、催办次数、上次催办
- 安全约束(为 Notifier 准备):
  - `dedupe_key` 生成函数:`{date}:{item_id}:{reminder_type}`(PRD §11.1 简化版,date 在 key 里就能天级幂等)
  - `check_rate_limit(welink_id)` — 单责任人当天上限检查
  - `check_run_limit(run_id)` — 单次运行上限
  - 这三个函数会被 M1-B 的 `send_welink` 内部强制调用
- `debug` 入口:`python -m pm_agent debug --tool read_excel` / `--tool query_state`

## 设计要点(强约束)
1. **每个 tool 都是 pure Python function**,签名清晰、返回 JSON-serializable dict,不依赖 LLM 即可单独跑。这是 P6 原则的根基。
2. **主表只读**:全程不写 `项目630流水线排期计划.xlsx`。tool 函数末尾应有 `assert excel_mtime == initial_mtime` 类的校验。
3. **稳定 itemId**:`normalize` 仅做空白压缩 + Unicode NFKC + 中文标点统一,不改语义。"序号"不参与 ID。
4. **WorkItem 字段对齐 PRD §15.1**,字段名 Python snake_case,但语义不变。
5. **SQLite WAL 模式**,允许并发读(虽然 MVP 单进程,留兜底)。
6. **不实现 LLM 相关代码** — 本 issue 只交付工具层,M2 才接 agent。
7. **不实现 Notifier** — Notifier 在 M1-B。
8. **不实现规则引擎** — 规则在 M1-B 作为 `query_rule_suggestions` 工具。

## 输入 / 输出
**输入:** `source/项目630流水线排期计划.xlsx`,`config/pm-agent.yaml`
**输出:**
- `python -m pm_agent debug --tool read_excel` 输出 JSON,含 179 条 WorkItem
- `state/pm-agent.db` 自动创建并建好四张表
- `python -m pm_agent debug --tool query_state` 能从空库查到空、写后能查到

## 验收标准
1. ✅ `read_excel` 解析 179 ±5 条
2. ✅ 主表 mtime 在工具运行前后不变
3. ✅ 同一 Excel 跑两次,所有 itemId 完全一致
4. ✅ 改一条事项的"用户问题原文",该 itemId 变化,旧 itemId 进入 `vanished_item_ids`

6. ✅ 状态映射:Open+待验收、Close+已完成、Open+开发中等 9 种已知组合全部不为"未知"
7. ✅ 首次跑自动创建 `state/pm-agent.db`,四张表 schema 与设计文档 §5 一致
8. ✅ `dedupe_key` / `check_rate_limit` / `check_run_limit` 三个安全函数有单元测试覆盖
